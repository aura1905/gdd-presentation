01 발주 배경

배럭 뷰에서 캐릭터들이 각자 배치된 시설(주방·공방·훈련장) 앞에서 실제 작업하는 모션을 재생합니다. 현재 전투 스프라이트(idle / attack / skill / run / die)에는 배럭 전용 작업 동작이 없으므로, 각 캐릭터별로 별도 work 스프라이트 시트 1장씩을 신규 발주합니다.

구현 방식: 기존 전투 스프라이트와 별도 파일 (work_sprite.png + work_frames.json). 시설에 배치된 캐릭터는 work 애니를 루프 재생하며 건물 앞 고정 위치에 서 있습니다. 배치되지 않은 캐릭터(휴식)는 기존 idle 유지.

02 공통 기술 규격

스프라이트 레이아웃
수평 스트립 (1행×N열). 기존 전투 스프라이트와 동일 방식
프레임 크기
캐릭터별 상이 (하단 표 참고). 기존 전투 스프라이트와 동일 캔버스 크기 사용
프레임 수
6~8 프레임 루프 (동작 내용에 따라 조정 가능)
프레임 지속 시간
기본 100ms/frame. 동작 강조 구간은 60ms, 정지 구간은 200~300ms
파일 포맷
work_sprite.png (PNG, 배경 투명) + work_frames.json (Aseprite 내보내기 호환)
frames.json 태그
"name": "work", "from": 0, "to": N-1, "direction": "forward"
캐릭터 정렬
bottom-center 기준 (발 위치 = 캔버스 하단 중앙). 기존 스프라이트와 동일
방향
오른쪽을 향함 (화면상 시설 방향). 기존 전투 스프라이트 기본 방향과 동일
배경
투명 (알파 채널). 소품/도구는 캐릭터 손 위에 그림 (별도 레이어 필요 없음)
납품 경로
demo/assets/sprites/{CharName}/work_sprite.png & work_frames.json
주의: 기존 sprite.png·frames.json은 수정하지 않습니다. work 스프라이트는 별도 파일로 납품하며 기존 전투 스프라이트와 독립적으로 동작합니다.
