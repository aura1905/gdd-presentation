PlayerTable

123행

게임에 등장하는 123명 플레이어 캐릭터의 원형(템플릿)을 정의하는 마스터 테이블. 각 캐릭터는 직업(F/S/M/W/L), 역할(Dealer/Tanker/Healer/Support), 소속 진영, 기본 스탯과 레벨당 성장률을 가진다. 전투 시 이 테이블의 Base 스탯에 레벨 성장률을 곱해 실전 스탯을 계산하며, SkillTable에서 해당 캐릭터 ID로 귀속된 고유 스킬(Slot 1~3)과 비전스킬(Slot 4~5)을 조회한다.

EnemyTemplateTable

108행

게임에 등장하는 108종 적 유닛의 원형(템플릿)을 정의하는 테이블. 10개 종족(야수·곤충·언데드 등) × 최대 16종으로 구성되며, 각 템플릿은 기본 스탯(HP/ATK/DEF/SPD)·스킬 목록·역할(Dealer/Tanker/Support/Leader)·최소 등장 강도를 가진다. 이 테이블 자체는 "설계도"이며, 실전 스탯은 헥스 레벨에 따른 스케일링 공식과 접두어(Dire/Alpha/Elder/King) 배수를 곱해 결정된다. EnemyPartyTable이 이 ID를 참조해 3인 파티를 편성한다.

필드타입설명

FamilycharB(야수)/I(곤충)/U(언데드)/H(인간)/D(악마)/E(정령)/C(구조물)/P(식물)/A(수생)/X(용족)
Tierint1~7 (강도 단계)

MinStrengthint최소 등장 리전 강도
ElementenumNone/Fire/Ice/Dark/Light/Nature/Water
AIValueint파티 편성 예산 단위

스탯 스케일링 공식: 실제스탯 = Base × (1 + (Lv-1) × Growth) × PrefixMult

EnemyPartyTable

154+행

플레이어가 헥스에서 조우하는 적 파티의 실제 편성표. 리전(RegionID) × 헥스 레벨(1~5) 조합마다 A/B/C 세 종류의 파티가 정의되어 있고, 전투 진입 시 이 중 하나가 랜덤 선택된다. 각 파티는 Slot1~3에 EnemyTemplateTable의 ID를 배치하며, 역할 조합(Balanced/Offensive/Defensive 등)에 따라 전투 체감 난이도가 달라진다. 리전의 주 적 종족이 우선 배치되고, 부 종족이 보조로 섞이는 구조.

필드타입설명
PartyIDint (PK)파티 고유 ID (1001~)
RegionIDint (FK)RegionTable 참조
HexLevelint헥스 레벨 (1~5)
PartyIndexint같은 Region+HexLevel 내 순번 (1=A, 2=B, 3=C)
PartyCountint해당 헥스레벨에서 등장하는 총 파티 수
Slot1~3string (FK)EnemyTemplateTable ID 참조
Slot1~3RoleenumDealer / Tanker / Support / Leader
FormationTypeenumBalanced / Offensive / Defensive / Leader / SingleRace

편성 유형

FormationType슬롯 구성특징
BalancedTanker + Dealer + Support안정적 구성. 가장 흔함.
OffensiveDealer + Dealer + Support높은 화력, 낮은 생존력.
DefensiveTanker + Tanker + Healer장기전 유도. 지구전 조합 필요.
LeaderLeader + Dealer + Dealer리더 버프로 화력 극대화.
SingleRace동일 종족 3체종족 시너지 보너스 발동.

파티 생성 규칙

1. 종족 우선: 리전의 EnemyFamilyMain이 Slot1~2에 배치, EnemyFamilySub가 Slot3에 배치.

2. 티어 필터: 해당 리전 Strength 이하의 MinStrength를 가진 템플릿만 후보군.

3. 역할 조합: FormationType에 맞는 Role 조합으로 슬롯 배정.

4. 다양성: 같은 Region+HexLevel에 A/B/C 3종을 두어 전투마다 다른 조합과 만남.

파티 ID 체계

PartyID = RegionID × 100 + HexLevel × 10 + PartyIndex 예: Region 5, HexLevel 3, 파티 B → PartyID = 5 × 100 + 3 × 10 + 2 = 532 범위: 1001 (R1-Lv1-A) ~ 42053 (R42-Lv5-C)

BondTable

85행

데미지 파이프라인

13단계

공격 1회가 최종 데미지로 변환되는 13단계 순차 연산. 주피증은 합연산, 받피감은 곱연산.

#단계공식비고
1기본 피해ATK × (ATK / (ATK + DEF×(1-PEN/100))) × 1.1base_damage_coefficient: 1.1
2스킬 배율× skill.multiplierSkillTable의 Multiplier
3HP 감쇠× (현재HP/최대HP)^0.3현재 비활성 (exponent=0.0)
4주피증× (1 + Σ buff%)합연산
5받피감× Π(1 - debuff%)곱연산
6속성 상성× 1.25 / 0.75 / 1.0화→풍→지→수→화, 광↔암
7역할 상성× 1.10 / 0.90 / 1.0Dealer→Support→Tanker→Dealer
8크리티컬× CRD (CRI 판정 성공 시)CRI cap 75%, CRD cap 300%
9피로도× fatigue_mult§피로도 참조
10난수× random(0.85~1.15)±15% 분산
11스킬뎀 보너스× (1 + skill_dmg_bonus%)리더 스킬 전용
12DR× (1 - DR/100)최종 피해 감소
13블럭× (1 - BRD/100)BLK 판정 성공 시. BLK cap 60%

C# 구현

public float CalculateDamage(Unit attacker, Unit defender, Skill skill) { float effAtk = attacker.ATK; float effDef = defender.DEF * (1f - attacker.PEN / 100f); // Stage 1: Base damage float dmg = effAtk * (effAtk / (effAtk + effDef)) * BASE_DMG_COEFF; // Stage 2: Skill multiplier dmg *= skill.Multiplier; // Stage 3: HP decay (disabled: exponent = 0) if (HP_DECAY_EXP > 0f) dmg *= Mathf.Pow(attacker.HPRatio, HP_DECAY_EXP); // Stage 4-5: Buff/Debuff dmg *= (1f + attacker.DamageIncrease); // 합연산 dmg *= (1f - defender.DamageReduce); // 곱연산 // Stage 6: Element advantage dmg *= GetElementMult(attacker.Element, defender.Element); // Stage 7: Role advantage dmg *= GetRoleMult(attacker.Role, defender.Role); // Stage 8: Critical if (Random.value < Mathf.Min(attacker.CRI / 100f, CRI_CAP)) dmg *= Mathf.Min(attacker.CRD / 100f, CRD_CAP); // Stage 9-10: Fatigue + Variance dmg *= GetFatigueMult(attacker.Fatigue); dmg *= Random.Range(0.85f, 1.15f); // Stage 11-12: Skill bonus + DR dmg *= (1f + attacker.SkillDmgBonus); dmg *= (1f - defender.DR / 100f); // Stage 13: Block if (Random.value < Mathf.Min(defender.BLK / 100f, BLK_CAP)) dmg *= (1f - defender.BRD / 100f); return Mathf.Max(dmg, MIN_DAMAGE); }

병종 상성 (5각)

JobClass

파티 리더의 JobClass(직업)를 기준으로 상성 판정. 5각 구조로 강약이 순환한다.

F(Fighter) → S(Scout) → M(Musketeer) → W(Wizard) → L(Leader) → F

상성 배수

관계ATK 배수DEF 배수예시
강세 (Strong)×1.25×1.15F → M
약간 강세×1.10×1.05F → S
중립×1.00×1.00F → F
약간 열세×0.95×0.97F → L
열세 (Weak)×0.90×0.93F → W

동일 병종 편성 보너스

같은 JobClass 수유니티 배수효과
3명 (풀 유니티)×1.5상성 효과 1.5배 증폭
2명×1.2상성 효과 1.2배 증폭
1명 (혼합)×1.0기본값

C# 구현

// 상성 매트릭스 [공격자, 방어자] → 배수 static readonly float[,] ATK_MATRIX = { // F S M W L /* F */ { 1.00f,1.10f,1.25f,0.90f,0.95f }, /* S */ { 0.90f,1.00f,1.10f,1.25f,0.95f }, /* M */ { 0.90f,0.95f,1.00f,1.10f,1.25f }, /* W */ { 1.25f,0.90f,0.95f,1.00f,1.10f }, /* L */ { 1.10f,1.25f,0.90f,0.95f,1.00f }, }; void ApplyJobAdvantage(Unit attacker, Unit defender, int sameJobCount) { int atkIdx = (int)attacker.JobClass; int defIdx = (int)defender.JobClass; float mult = ATK_MATRIX[atkIdx, defIdx]; float unity = sameJobCount == 3 ? 1.5f : sameJobCount == 2 ? 1.2f : 1.0f; attacker.ATK *= 1f + (mult - 1f) * unity; }

Element

火(Fire) → 風(Wind) → 地(Nature) → 水(Water) → 火

光(Light) ↔ 闇(Dark) — 상호 유리

관계배수조건
유리×1.25순환 순서상 공격자 우위
불리×0.75순환 역방향
중립 / 동일×1.00같은 속성이거나 None
static float GetElementMult(Element atk, Element def) { if (atk == Element.None || def == Element.None) return 1f; if (atk == def) return 1f; // 광↔암 상호 유리 if ((atk == Element.Light && def == Element.Dark) || (atk == Element.Dark && def == Element.Light)) return 1.25f; // 화→풍→지→수→화 순환 var cycle = new[] { Element.Fire, Element.Wind, Element.Nature, Element.Water }; int atkIdx = Array.IndexOf(cycle, atk); int defIdx = Array.IndexOf(cycle, def); if (atkIdx < 0 || defIdx < 0) return 1f; return (atkIdx + 1) % 4 == defIdx ? 1.25f // 유리 : (defIdx + 1) % 4 == atkIdx ? 0.75f // 불리 : 1f; }

CC 면역 규칙

우선순위: 이미 CC 중일 때, 같거나 높은 우선순위의 CC만 덮어쓰기 가능.
