게임 개요

CORE CONCEPT

장르: 가문 경영 전략 RPG (풋볼매니저 × 삼국지전략판 × HoMM × GE IP)

핵심 컨셉: GE 원작의 395명 캐릭터를 영입·육성·편성하여 신대륙에서 가문을 경영하고 세력을 확장하는 전략 게임.

배경: 레콩키스타 시대(1522~) — 3년 전쟁 이후 신대륙 개척 경쟁이 격화된 시기.

🎯 핵심 설계 철학 — "가문 경영 서사"

캐릭터 = 전투 도구가 아닌, 가문의 식구 영입하고 → 같이 살고 → 역할을 주고 → 밥을 먹이고 → 관계가 형성되고 → 함께 전쟁하고 → 때론 떠나보내고 → 가문의 역사가 쌓이는 것 GE 원작의 혁신적 MCC(다캐릭터 동시 조작) + 395명 영입 캐릭터 + 가문 시스템의 정신을 계승하되, 삼전/FM/HoMM의 체계적 시스템 적용.

🎮 핵심 설계 원칙

원칙 설명
전투보다 운영 전투는 결과를 보는 것. 운영(편성/배치/경영)이 핵심 재미.
캐릭터 = 가문의 식구 도구가 아닌 관계의 대상. 충성도·사기·관계가 성과에 영향.
레벨 깡패 없음 조합·상성·시너지가 승패 결정. 약해도 상성 좋으면 이긴다.
캐릭터는 항상 부족 공격 + 수비 + 토벌 + 생활 동시에 → 더 영입, 더 육성.
PvF (Player vs Faction) PvP 피로감 없는 NPC 세력 전쟁. 적의 적은 친구.
감정적 과금 "저 세력을 꺾으려면 이 캐릭터가 필요" → 전략적 필요에 의한 과금.

🔁 의미적 포함 관계 (TARGET_IMPLIES, 자동 fan-out)

한 행동이 여러 quest 타입을 동시에 진행시킴. 예: "첫 점령" 했는데 "첫 토벌" quest가 그대로면 어색.

siege_gate ──┐
siege_fort ──┼──▶ occupy ──▶ subjugate
             │
             └──▶ siege_any
  • occupy ⊃ subjugate: 점령 = 적 격파 + 헥스 소유권. 점령 시 토벌 quest 자동 진행
  • siege_gate/fort ⊃ occupy + siege_any: 구조물 함락 = 점령의 한 종류, 함락 카운트도 누적
  • 역방향 X: 토벌만으론 점령 quest 진행 X (헥스 소유권 변동 없음)

구현: engine/quests.js의 TARGET_IMPLIES 매핑. reportProgress()가 BFS로 펼침 (cycle 방지).
