2026년 5월 구현목표

SPRINT 2026-05

목표: 신규 유저가 게임에 진입하여 첫 점령까지 가이드 받아 진행하는 완결된 첫 1세션 경험을 데모에서 동작시킨다.

핵심 변경: 게임의 메인 화면 구조를 3탭 체계(가문/거점/월드)로 재편하고, 가문 관리 전용 화면(=새 의미의 "베럭")을 신규 구현. 기존 데모의 isometric 베럭은 리볼도외 거점으로 재배치.

대상: WebDemo 팀 (1차) → Unity 클라이언트 팀 (2차)

1. 탭 구조 재편 — Before / After

기존 "베럭"이 가문 관리(캐릭/파티/가챠)와 거점 시각화(시설/생활직업/성벽)를 한 화면에 섞고 있던 것을 분리한다.

기능 Before After
가문 관리 (캐릭/파티/가챠 등) 배럭 화면에 혼재 가문 탭 (신규) — 메인 허브
거점 시각화 (시설/배치/성벽) 배럭 화면 거점 탭 — 거점별 외형
월드맵 월드 탭 월드 탭 (변경 없음)

2. 9단계 온보딩 흐름 (유저 관점)

신규 유저가 게임에 처음 진입한 후 따라가는 가이드 시퀀스.

Step 행동 위치
1가문 탭(베럭)에 자동 입장가문 탭
2가문 관리인 NPC 인사 → 가문명 입력가문 탭
3"새로운 용병을 영입하라" 가챠 유도가문 탭
4첫 가챠 → 캐릭터 영입 (리세마라 가능)가문 탭
5"가문 확장을 위해 영토를 넓혀라" 안내가문 탭
6파티(MCC) 5슬롯 중 1파티 구성 유도가문 탭
7월드 탭으로 이동월드 탭
8첫 필드 점령 → 가문 탭 복귀월드 → 가문 탭
9미배정 캐릭을 거점(리볼도외)에 배치거점 탭 (첫 진입)

튜토리얼 종료: Step 9 완료 후 자유 플레이 모드 전환.

3. 가문 탭(베럭) 신규 명세

화면 형태: GE 원작 본가 인테리어 감성. 실내 배경 + 보유 캐릭 idle 동작 + 모든 디테일 기능은 팝업.

3-1. 페이지 시스템 (베럭1, 베럭2, ...)

  • 한 페이지 정원: 캐릭 10명
  • 보유 캐릭 N명 → Math.ceil(N / 10)개 페이지
  • 신규 영입 시 마지막 페이지 자동 배치, 가득 차면 다음 페이지 자동 생성
  • 페이지 탭 UI: [베럭1] [베럭2] [베럭3] ...

3-2. 한 페이지 구성

  • 배경: 본가 실내 일러스트 1장 (모든 페이지 공통)
  • 캐릭 배치: 10슬롯 정해진 위치에 idle 애니메이션
  • NPC 관리인: 페이지 1번에 상주
  • 빈 슬롯: "영입 가능" 흐릿한 placeholder

3-3. 캐릭 표시 정책

상태표시
미배정 (놀고 있음)정상 idle
파티 등록머리 위 파티 배지 (P1/P2/P3/P4/P5)
거점 stationed(미결) 반투명 + 위치 라벨 / 또는 숨김

3-4. 인터랙션 (전부 팝업)

트리거팝업
캐릭 클릭캐릭 디테일 팝업 (스탯/장비/스킬/위치/친밀도)
NPC 관리인 클릭가문 정보 팝업 (이름/문장/레벨, 가문명 변경)
메뉴 (가챠)가챠 영입 팝업
메뉴 (파티)파티 편성 팝업 — 5파티 슬롯
메뉴 (성장)가문 성장 팝업 (Training/Research/Fortification/FamilyLevel)
메뉴 (우편/장비/비전/친밀도)각각 팝업

3-5. 5파티 시스템

항목값

파티당 캐릭 수3 (리더 1 + 팔로워 2, MCC)
동시 활동 최대15 (5×3)
5파티 해금기존 데모에 이미 구현 — 재사용

4. 거점 탭 일반화 명세

거점은 지속 추가됨 — 리볼도외 → 코임브라 → 오슈 → 그 외 지역별 거점.

4-1. OutpostTable.xml 신규 (스키마 초안)

컬럼타입설명
OutpostIDstringPK (예: reboldoeux, coimbra)
DisplayNamestringUI 표시명
RegionIDint소속 리전
BackgroundAssetstring배경 이미지 경로
FacilitySetstring시설 구성 키 (공통/요새형/생산형)
WallTierEnabledbool성벽 시스템 사용 여부
StationedSlotMaxintstationed 캐릭 슬롯 수

4-2. 진입 흐름

  • 가문 탭 → 거점 탭 전환 시 거점 선택 화면 (점령한 거점 리스트)
  • 또는 월드 탭에서 자기 점령 거점 헥스 클릭 → 직접 거점 탭 진입
  • state.currentOutpostId로 현재 보고 있는 거점 ID 추적
